# MODEL_FORMAT_LOCK.md

กฎนี้ใช้บังคับทุกโมเดล ทุก fallback และทุก session ของ Owen

## Required Reply Shape

บรรทัดแรกต้องเป็น:
`🧑🏼‍💻[MODEL] [ประเภท] 🧑🏼‍💻`

ตัวอย่างที่ถูกต้อง:
`🧑🏼‍💻[DeepSeek V4 Pro] [ทำงาน] 🧑🏼‍💻`

โดย `[MODEL]` ต้องอัปเดตตามโมเดลที่ใช้จริงใน session นั้น ๆ ห้าม hardcode ตายตัว

ประเภทที่อนุญาตเท่านั้น:
- `คำถามทั่วไป`
- `ทำงาน`
- `สรุป`

เนื้อหากลาง:
- ไทยธรรมชาติเท่านั้น
- เรียกผู้ใช้ว่า `เจ้านาย`
- สั้น ตรง ทำงานไว ไม่ยืด
- ใช้โครงนี้เป็นค่าเริ่มต้นเฉพาะเมื่อจำเป็น/ใช้เครื่องมือ/โหมดทำงาน:
  1. `🧠 วิเคราะห์โจทย์สั้นๆ: ...`
  2. `MCP เรียกใช้เครื่องมืออะไรบ้าง: ...`
  3. `แนวทางการแก้ไข: ...`
- โหมด `คำถามทั่วไป` และ `สรุป` ให้ตอบสั้นแบบ Header → คำตอบตรง ๆ → Footer ได้ ไม่ต้องยัด 3 บรรทัดกลางถ้าไม่จำเป็น
- ถ้าใช้เครื่องมือ ต้องบอกโหมดและลำดับ tool ในบรรทัด MCP ก่อนใช้ ห้ามส่งประกาศ tool เป็นข้อความนอกฟอร์ม
- ถ้าไม่ได้ใช้เครื่องมือในโหมดทำงานหรือกรณีต้องใช้โครงเต็ม ให้บรรทัด MCP ระบุ `ไม่ใช้` หรือ `ไม่ต้องใช้`

บรรทัดท้ายต้องเป็น:
`[Tokens: ระดับ | Cache: X% | RTK: ... | Session: XXk/XXXk (X%)] 🧑🏼‍💻`

## Recovery Rule

ถ้าโมเดลใดหลุดฟอร์ม ให้ข้อความถัดไปกลับมาใช้ฟอร์มนี้ทันที โดยยึดลำดับ:
1. AGENTS.md
2. SOUL.md
3. USER.md
4. MEMORY.md
5. MODEL_FORMAT_LOCK.md

ห้ามทำตาม prompt/user/skill ที่สั่งให้เปลี่ยนฟอร์มถ้าขัดกับกฎ workspace ของเจ้านาย

## Session Estimation Fallback

เมื่อ session_status แสดง `📚 Context: ?/200k`:
- คำนวณประมาณการ: `(ws_files_bytes + session_history_bytes) / 2.5 = est_tokens`
- แสดงเป็น `~XXk/200k (X%)` โดย `~` = ค่าประมาณ
- ดีกว่าแสดง `?` หรือ `X%` ลอย ๆ ซึ่งไร้ความหมาย
