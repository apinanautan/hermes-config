
## Hermes gateway shutdown loop fix
- เจ้านายอนุญาตให้ตรวจ/แก้ Hermes หลังเจอข้อความ `Gateway shutting down — Your current task will be interrupted` วน
- พบ 2 สาเหตุจาก journal:
  1. มีคำสั่งจากภายใน Hermes เองสั่ง `systemctl --user restart hermes-gateway` / `pkill ... gateway` หลายครั้ง ทำให้งานที่กำลังตอบถูกตัด
  2. `hermes-gateway-comfyui.service` เปิด `Restart=always` แต่ profile `comfyui-bot` ไม่มี จึง fail แล้ว auto-restart ทุก 5 นาที (restart counter 95)
- แก้โดยปิด service ที่เสีย: `systemctl --user disable --now hermes-gateway-comfyui.service`
- ตรวจแล้ว Hermes หลัก `hermes-gateway.service` ยัง active/running และ health endpoint `http://127.0.0.1:8642/health` ตอบ `{"status":"ok","platform":"hermes-agent"}`
- เพิ่มเติม 09:11: ภาพจาก Telegram ยืนยันว่า Mes_bot/Hermes ตอบ `เอาน่ะ...ผมรีสตาร์ทเลย` แล้วรัน `systemctl --user restart hermes-gateway` หลังผู้ใช้พิมพ์ `มายัง`; session Hermes หลักมี `resume_pending: true` จาก `restart_interrupted` จึง resume งานเก่าแล้วสั่ง restart ซ้ำ
- แก้ถาวรโดย backup `~/.hermes/config.yaml`, ลบ allowlist ที่อนุญาตคำสั่ง restart/kill Hermes gateway ออกจาก `command_allowlist`, ตั้ง `resume_pending=false` ใน `~/.hermes/sessions/sessions.json`, restart Hermes หนึ่งครั้งเพื่อโหลด config ใหม่; health OK และ `hermes-gateway-comfyui.service` ยัง disabled/inactive
