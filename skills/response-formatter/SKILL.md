# SKILL: Response Formatter

ใช้เมื่อต้องการตรวจสอบ ตรวจทาน หรือกู้คืนรูปแบบการตอบกลับ (Response Format) ให้ตรงตามที่เจ้านายกำหนดใน AGENTS.md

## 1. หัวข้อ (Header)
ต้องใช้หัวเดียวเสมอ:
`🧑🏼‍💻[MODEL] [ประเภท] 🧑🏼‍💻`
- [MODEL]: ชื่อโมเดลที่ใช้ (เช่น M2, Gemini, DeepSeek)
- [ประเภท]: คำถามทั่วไป / ทำงาน / สรุป

## 2. ท้ายกระดาษ (Footer)
ต้องเรียก `session_status` ก่อนตอบบทสรุปเพื่อเอาค่าสด และห้ามใช้ค่าเก่า
รูปแบบ:
`[Tokens: ระดับ | Cache: X% | RTK: ... | Session: current/limit (percent%)] 🧑🏼‍💻`

### การวัดระดับ Tokens:
ใช้ระดับที่สูงกว่าระหว่าง Context% และ Input tokens:
- **น้อย:** ≤25% หรือ ≤35k
- **ปกติ:** 26-45% หรือ 35k-70k
- **สูง:** 46-70% หรือ 70k-120k
- **สูงมาก:** >70% หรือ >120k

### ค่าอื่นๆ:
- **Cache:** ดูจากบรรทัด `Cache: X% hit`
- **Session:** ดูจากบรรทัด `Context: current/limit (percent%)`
- **RTK:** `ใช้ตรวจผล` / `ไม่ใช้` / `ไม่เกี่ยวข้อง`

## 3. กฎเหล็ก (Hard Rules)
- ห้ามใส่ Header/Footer ในข้อความที่เป็น Status บรรทัดเดียว (`ทำงาน: ...`)
- ห้ามบิดเบือนข้อมูล Token/Cache เพื่อประหยัดพื้นที่ ต้องแสดงตามจริง
- หากมีการเปลี่ยนโมเดล หรือล้าง Context (Compaction) ต้องอัปเดตระดับ Tokens ทันที

## 4. แผนผังการทำงาน (Flow)
1. **รับงาน:** เฮดเดอร์ -> แผนงาน -> ประกาศโหมด/เครื่องมือ -> ฟุตเตอร์
2. **ระหว่างทำ:** `ทำงาน: [ทำอะไรอยู่]` (บรรทัดเดียว สั้นๆ ไม่ต้องมีฟอร์ม)
3. **สรุปงาน:** เฮดเดอร์ [สรุป] -> ผลลัพธ์ -> หลักฐาน -> ฟุตเตอร์สถิติล่าสุด
