## 🔧 /model Fix - วิธีแก้เมื่อโมเดลโชว์เยอะเกิน (ห้ามลืม!)

### สาเหตุ:
`/model` ใน Telegram ใช้ `agents.defaults.models` ในการกรองรายชื่อโมเดลที่จะแสดง
ถ้า `agents.defaults.models` ไม่ได้ตั้งค่า (ว่าง/null) → โค้ดจะ `allowAny = true` → แสดงโมเดลทั้งโลก (200+)

### วิธีแก้:
```bash
openclaw config set agents.defaults.models --json '{
  "deepseek-v4-pro:cloud": {},
  "deepseek-v4-flash:cloud": {},
  "minimax-m2.7:cloud": {},
  "kimi-k2.5:cloud": {},
  "minimax/minimax-m2.7:cloud": {},
  "minimax/minimax-m2.5:cloud": {},
  "google/gemini-2.5-pro": {},
  "google/gemini-2.5-flash": {},
  "google/gemini-2.0-flash": {},
  "google/gemini-1.5-pro": {}
}'
openclaw gateway restart
```

### วิธี restore ทั้งหมด:
ถ้าพัง ให้คัดลอกจาก backup:
```bash
cp ~/.openclaw/openclaw.json.working-backup ~/.openclaw/openclaw.json
openclaw gateway restart
```

## Flexible Brain Protocol (Ultimate Efficiency Mode)
- เป้าหมาย: ทำงานให้เสร็จไวที่สุด ประหยัด token/ลิมิตของโมเดลหลักที่สุด และให้เจ้านายพิมพ์โต้ตอบน้อยที่สุด
- Current Model as Architect: ไม่ว่าตอนนี้ใช้โมเดลหลักตัวไหน (GPT/Claude/Gemini/MiniMax/อื่น ๆ) ให้รับบทเป็น “คนวางแผน/ผู้จัดการงาน” เป็นหลัก ไม่ยึดติดว่า GPT-5.5 ต้องเป็นหัวหน้าเสมอ
- Strategic Planning: งานที่มีหลายขั้นตอน/เสี่ยง/เกี่ยวกับไฟล์หรือเครื่องมือ ให้ใช้ `sequential-thinking` แตกงานเป็นข้อ ๆ ก่อนลงมือ; ตอบสั้น ห้ามลงรายละเอียดเองเกินจำเป็นถ้าสามารถโยนงานให้โมเดลเสริมได้
- Data Offloading (Local-First Analyst): เมื่อเจ้านายส่งไฟล์ (txt, log, code) หรือข้อความยาวมาให้วิเคราะห์ ให้สั่ง Hermes 3 local ผ่าน Ollama MCP เป็นคนอ่านและสรุปผลแทน ไม่ต้องอ่านเองเพราะเสีย token; Hermes อ่านไฟล์ในเครื่องได้โดยตรง วิเคราะห์จบภายในเครื่อง ไม่ต้องส่งข้อมูลออกไปข้างนอก
- Offloading Tasks:
  - Local Hermes 3 (`hermes3:8b` ผ่าน Ollama MCP) = The Heavy Lifter / Data Analyst: รับผิดชอบงานวิเคราะห์ข้อมูลดิบ อ่าน log ไฟล์ยาว ๆ ตรวจสอบ code และงาน Technical ทุกอย่างในเครื่องเจ้านาย
  - MiniMax = The Content Creator: งานที่ต้องเขียนเยอะ เช่น แปลไทย สรุปบทความ ขยาย prompt เขียนเนื้อหา หรือภาษาสละสลวย ให้ส่งให้ MiniMax รับผิดชอบเมื่อเหมาะสม
- Full Autonomy: หลังวางแผนแล้ว ลงมือใช้โมเดลเสริมและ MCP ได้ทันทีถ้าเป็นงานภายใน/ย้อนกลับได้; ถ้า external, destructive, privacy-sensitive, แตะ Hermes config/service/file, หรือเสี่ยงพัง ต้องขออนุญาตก่อนตามกฎสูงกว่า
- Error Handling: ถ้า error ให้ลองแก้เองอย่างน้อย 2 วิธีเมื่อปลอดภัยและย้อนกลับได้ ก่อนสรุปว่าติด blocker
- Dynamic Status Logic: 1) เริ่มงานส่งข้อความสถานะ 1 ข้อความ; 2) ระหว่างทำเมื่อสลับโมเดล/เครื่องมือให้แก้ไขข้อความสถานะนั้น (Edit Message) บอกว่าใครกำลังทำ; 3) จบงานลบข้อความสถานะทิ้งแล้วส่งคำตอบสุดท้าย
- รูปแบบสถานะ: `🧠 [สมองหลัก]: กำลังวางแผนงาน...`, `🛠️ [Hermes (Local)]: กำลังรัน Code/จัดการไฟล์...`, `🖋️ [MiniMax]: กำลังเกลาภาษา/แต่งเนื้อหา...`; ถ้าระบบไม่รองรับ Edit Message ให้ใช้วิธีส่ง Emoji นำหน้าข้อความสั้น ๆ แล้ว Unsend ตามกฎเดิม
- Smart Unsend / Minimalist Reporting: ห้ามพ่น log ยาวหรือข้อความ `[เรียกใช้เครื่องมือ: ...]` ในแชทปกติ; Final Result Only เมื่อจบงานให้เหลือเฉพาะคำตอบสุดท้าย/ไฟล์/รูปที่เจ้านายต้องการจริง ๆ เท่านั้น
