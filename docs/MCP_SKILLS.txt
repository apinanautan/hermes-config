# OpenClaw MCP & Skills - Complete Reference

## Prompt Source Dedup
- Mirror-only: local worker ห้าม inject docs mirror family ทั้งหมดพร้อม root copy
- Cloud planner inject ได้แบบ source เดียวต่อหมวด และต้อง dedupe ด้วย `source_id + checksum`
- REQDBG breakdown: `root_rules`, `docs_mirror`, `memory`, `tools`, `mcp_schema`

## MCP Servers (ทำงานแล้วตอนนี้)

### 1. socraticode
- Command: `npx -y socraticode`
- Description: สกิลวิเคราะห์โค้ดหลัก ต้องใช้ก่อนแก้ไขโค้ดทุกครั้ง
- ต้อง load SKILL.md ก่อน: `~/.openclaw/plugin-skills/socraticode-code/SKILL.md`

### 2. puppeteer
- Command: `npx -y @kirkdeam/puppeteer-mcp-server`
- Description: Browser automation ควบคุม Chrome ผ่าน CDP

### 3. sequential-thinking
- Command: `/home/apinan/.openclaw/mcp-node/node_modules/.bin/mcp-server-sequential-thinking`
- Description: แตกงานซับซ้อนเป็นข้อๆ วางแผนก่อนลงมือ

### 4. filesystem
- Command: `/home/apinan/.openclaw/mcp-node/node_modules/.bin/mcp-server-filesystem`
- Allowed dirs:
  - `/home/apinan/.openclaw/workspace`
  - `/mnt/c/Users/[ถามรหัสจากเจ้านายเมื่อจำเป็น]/Downloads`
  - `/mnt/c/Users/[ถามรหัสจากเจ้านายเมื่อจำเป็น]/Pictures`
- Description: อ่าน/เขียน/ค้นหาไฟล์ในเครื่อง

### 5. ollama
- Command: `/home/apinan/.openclaw/mcp-node/node_modules/.bin/ollama-mcp-server`
- Env: `OLLAMA_HOST=http://127.0.0.1:11434`
- Description: รันโมเดล local (Hermes 3, nomic-embed-text)
- ใช้สำหรับ: Data offloading, วิเคราะห์ไฟล์/โค้ดแทนโมเดลหลัก

## OpenClaw Tools (Native - ไม่ใช่ MCP)

- exec: รัน shell command
- read/read_text_file: อ่านไฟล์
- write/edit: เขียน/แก้ไฟล์
- filesystem__* : จัดการไฟล์ (MCP)
- image: วิเคราะห์/สร้างรูป
- web_search/web_fetch: ค้นหาเว็บ
- cron: ตั้งเวลางาน
- sessions_* : จัดการ session/sub-agent
- subagents: สั่งงาน sub-agent
- ollama__* : รันโมเดล local
- socraticode__* : วิเคราะห์โค้ด
- puppeteer__* : browser automation
- memory_* : ค้นหาความจำ
